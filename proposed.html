ARU — Incorporating AI Agents into Scrum (Before vs After)

ARU houses deterministic code for claim adjudication. Currently, business analysts expand manual auditor instructions using SOPs, references, and data sheets into detailed processing steps and flow diagrams with help from junior staff and systems analysts before handing them to developers. In the new process, AI agents generate the first complete artifact set (elaborated instructions, system considerations, and flow diagrams) directly from analyst inputs, and a BPM workflow manages validation, regeneration, and analyst–developer collaboration within the sprint lifecycle.

AI Integration into Scrum

  • Backlog Refinement / Definition of Ready: Analyst provides instructions and reference sources → AI generates complete processing artifacts → Analyst and Systems Analyst validate → story becomes ready for sprint.
  • Sprint Execution: Developers implement deterministic code using validated artifacts and coding agents, with BPM-managed collaboration loops for clarifications.
  • Sprint Review & Acceptance: Functionality is verified against the AI-generated, analyst-approved processing instructions and diagrams.

This approach reduces manual artifact creation effort, accelerates readiness of user stories, and embeds AI agents directly into the Scrum delivery lifecycle while maintaining deterministic implementation control within ARU.
